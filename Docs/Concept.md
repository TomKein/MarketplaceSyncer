# Концепция MarketplaceSyncer

## Общее описание
Сервис для синхронизации данных между различными маркетплейсами.
Реализован как Worker Service (HostedService), работает постоянно в фоне.

## Технологический стек
- **Платформа**: .NET 10
- **Тип приложения**: Worker Service
- **Управление пакетами**: Central Package Management (CPM) via `Directory.Packages.props`
- **База данных**: PostgreSQL (через `Npgsql`, `linq2db`)
- **API**: `Microsoft.Extensions.Http` для внешних вызовов

## Функциональные требования
1.  **Постоянная работа**: Сервис выполняется как фоновый процесс.
2.  **База данных**: Прямой доступ к собственной БД.
3.  **Интеграции**: Выполнение API-вызовов к внешним сервисам (маркетплейсам).
4.  **Интерфейс**: На текущем этапе Web или API интерфейс отсутствует.

## Архитектурные решения
- Использование шаблона Worker Service для обеспечения постоянной работы.
- Централизованное управление версиями пакетов для упрощения поддержки зависимостей.
- Отсутствие UI/API endpoints в первой версии (фокус на бэкенд логике).
